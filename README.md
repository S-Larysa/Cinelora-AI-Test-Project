# Cinelora AI - QA Portfolio Project 🚀

## 📋 Project Overview
This repository contains a comprehensive QA documentation suite and bug-tracking emulation for **Cinelora AI**, a conceptual multimodal AI generation platform. 

> **Note:** This is a fictional project created for portfolio purposes to demonstrate end-to-end software testing processes, from planning and strategy to execution and defect management.

## 🛠 Documentation Included
In this repository, you can find the following QA artefacts:
- **[Test Strategy](Test_Strategy_CINELORA_AI.md)**: High-level approach and testing types for the platform.
- **[Test Plan](Test_Plan_CINELORA_AI.md)**: Detailed scope, resources, and schedule for the testing cycle.
- **[Estimation](Estimation_CINELORA_AI.md)**: Time and resource calculations.
- **[Traceability Matrix](Traceability_Matrix_CINELORA_AI.csv)**: Mapping business requirements to test cases to ensure full coverage.

## 🐞 Bug Tracking & Management
I use **GitHub Issues** and **GitHub Projects** to manage the defect lifecycle, simulating a professional Agile environment:
- **[Active Bug Reports](https://github.com/S-Larysa/Cinelora-AI-Test-Project/issues)**: Detailed reports with reproduction steps, severity, and priority.
- **[Agile Management Board](https://github.com/users/S-Larysa/projects/2)**: Real-time tracking of tasks and bug statuses (In Progress, Testing, Done).
- **Prioritisation**: Defects are categorised using labels like `S1: Blocker`, `P1: Critical`, and `Security` to demonstrate triage skills.

## 🧪 Skills Demonstrated
- **Test Design**: Creating structured test documentation from scratch.
- **Defect Management**: Reporting and managing bugs within a tracking system.
- **Agile Methodology**: Working with Story Points and status-based workflows.
- **Tools**: GitHub (Issues/Projects), Markdown, Git.
